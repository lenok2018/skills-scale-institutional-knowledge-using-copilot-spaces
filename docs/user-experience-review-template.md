## User Experience Criteria

### Usability

#### Clarity and Understanding
- [ ] User goals and tasks are clear and well-defined
- [ ] Interface labels and instructions are clear and unambiguous
- [ ] Error messages are helpful and actionable
- [ ] User is never confused about what to do next
- [ ] Feature aligns with user mental models and expectations

**Notes/Issues**: 
_____________________________

#### Efficiency
- [ ] Common tasks can be completed quickly
- [ ] Users can achieve their goals with minimal clicks/steps
- [ ] No unnecessary friction or redundant steps
- [ ] Keyboard shortcuts available for power users (where appropriate)
- [ ] Loading states and progress indicators are appropriate

**Notes/Issues**: 
_____________________________

#### Error Prevention and Recovery
- [ ] Design prevents errors where possible (constraints, validation)
- [ ] Clear confirmation for destructive actions
- [ ] Easy recovery from errors (undo, clear error recovery paths)
- [ ] Inline validation provides immediate feedback
- [ ] Users can save progress and return later (where applicable)

**Notes/Issues**: 
_____________________________

---

### Accessibility

- [ ] Color contrast meets WCAG 2.1 AA standards (minimum 4.5:1 for text)
- [ ] All interactive elements are keyboard accessible
- [ ] Focus indicators are visible and clear
- [ ] Screen reader compatible with proper ARIA labels
- [ ] Images have meaningful alt text
- [ ] Form inputs have associated labels
- [ ] Heading hierarchy is logical and semantic
- [ ] No content conveyed by color alone
- [ ] Text is resizable without breaking layout
- [ ] Motion can be reduced/disabled for users with vestibular disorders

**Notes/Issues**: 
_____________________________

---

### Visual Design

#### Consistency
- [ ] Design follows established design system and component library
- [ ] Typography is consistent with brand and hierarchy standards
- [ ] Spacing and layout follow grid system
- [ ] Icons and imagery align with design language
- [ ] Interaction patterns match existing product behaviors

**Notes/Issues**: 
_____________________________

#### Visual Hierarchy
- [ ] Most important elements are visually prominent
- [ ] Clear distinction between primary and secondary actions
- [ ] Proper use of white space for scannability
- [ ] Related elements are visually grouped
- [ ] Visual flow guides user through intended path

**Notes/Issues**: 
_____________________________

---

### Content and Microcopy

- [ ] Tone and voice align with brand guidelines
- [ ] Content is concise and scannable
- [ ] Technical jargon is minimized or explained
- [ ] Calls-to-action are clear and compelling
- [ ] Empty states provide helpful guidance
- [ ] Success messages confirm user actions
- [ ] Help text available where needed

**Notes/Issues**: 
_____________________________

---

### Responsiveness and Performance

- [ ] Design works across target screen sizes and devices
- [ ] Touch targets are appropriately sized for mobile (minimum 44x44px)
- [ ] Layout adapts gracefully to different viewports
- [ ] Performance considerations addressed (lazy loading, image optimization)
- [ ] Perceived performance is good (skeleton screens, optimistic UI)
